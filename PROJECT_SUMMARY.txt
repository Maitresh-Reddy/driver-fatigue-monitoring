💡 KEY FEATURES
─────────────────────────────────────────────────────────────────────────────

1. ADAPTIVE BASELINE CALIBRATION
   • First ~10 seconds: learns driver-specific behavior
   • Measures: blink rate, head position, mouth movement
   • Continuously adapts for slow drift correction

2. COMPOSITE FATIGUE SCORING
   • Formula: 0.4×EyeClosure + 0.3×Yawn + 0.3×HeadDroop
   • Score range: 0-100
   • Thresholds: Low(20), Moderate(40), High(60), Critical(80)

3. CONTEXT-AWARE CLASSIFICATION
   • ALERT: score < 20 (normal driving)
   • DISTRACTED: 20 < score < 60 (some fatigue signs)
   • FATIGUED: score > 60 (multiple indicators)

4. FALSE ALERT REDUCTION
   • Multi-condition validation (multiple factors required)
   • Temporal smoothing (5-frame majority voting)
   • Alert cooldown (5 seconds between alerts)
   • Baseline comparison (individual differences handled)

5. EXPLAINABLE ALERTS
   • JSON logs with timestamp
   • Reasons provided
   • Example: "Eye closure for 2.3 seconds | 3 yawns detected | Critical score 78.5"

6. REAL-TIME MONITORING
   • 20-30 FPS on standard laptop
   • Face detection 95%+ rate
   • False alert rate <5%
   • Baseline calibration ~10 seconds

🎯 SYSTEM ARCHITECTURE
─────────────────────────────────────────────────────────────────────────────

Input (Video)
    ↓
Face Detection (MediaPipe)
    ↓
Facial Landmarks (468 points)
    ↓
Feature Extraction:
  • Eye regions + EAR (Eye Aspect Ratio)
  • Mouth region + MAR (Mouth Aspect Ratio)
  • Head pose angles (pitch, yaw, roll)
    ↓
CNN Inference:
  • Eye Model: predict eye state
  • Yawn Model: predict yawning
    ↓
Baseline Calibration:
  • Compare against driver-specific baseline
    ↓
Fatigue Scoring:
  • Compute 0-100 composite score
    ↓
State Classification:
  • Alert / Distracted / Fatigued
    ↓
Alert System:
  • Check conditions
  • Generate explanations
    ↓
Output:
  • Real-time visualization (video with overlays)
  • Alert logging (JSON with reasons)
  • Performance metrics (FPS, inference time)
